    • 接口(interface)定义了一个对象的行为规范，只定义规范不现实，具体的对象来实现规范的细节；

    # 一、接口类型

    • 在Go语言中(interface) 是一种类型，一种抽象的类型；
    • interface 是一组 method 的集合，是duck-type programming 的一种体现，接口做的事情就像是定义一个协议(规则)，只要一台机器有洗衣服和甩干的功能，我们就称为它是洗衣机，不关心属性(数据)，只关心行为(方法)；

    # 二、为什么要使用接口

    type Cat struct{}

func (c Cat) Say() string { return "喵喵喵" }

type Dog struct{}

func (d Dog) Say() string { return "汪汪汪" }

func main() {
	c := Cat{}
	fmt.Println("猫:", c.Say())
	d := Dog{}
	fmt.Println("狗:", d.Say())
}

    • 上面的代码中定义了猫和狗，然后它们都会叫，你会发现main函数中明显有重复的代码，如果我们后续再加上猪、青蛙等动物的话，我们的代码还会一直重复下去。那我们能不能把它们当成“能叫的动物”来处理呢？
    • 像类似的例子在我们编程过程中会经常遇到：
      • 比如一个网上商城可能使用支付宝、微信、银联等方式去在线支付，我们能不能把它们当成“支付方式”来处理呢？
      • 比如三角形，四边形，圆形都能计算周长和面积，我们能不能把它们当成“图形”来处理呢？
      • 比如销售、行政、程序员都能计算月薪，我们能不能把他们当成“员工”来处理呢？
    • Go语言中为了解决类似上面的问题，就设计了接口这个概念。接口区别于我们之前所有的具体类型，接口是一种抽象的类型。当你看到一个接口类型的值时，你不知道它是什么，唯一知道的是通过它的方法能做什么。
    • 没有接口类型的局限性示例如下：

type dog struct{}

type person struct{}

	fmt.Println("哈哈哈~")

// 接收一个参数，传递进来什么，我就打什么
func da(x cat) {
	x.speak()

func main() {
	var c1 cat
	// var d1 dog
	// var p1 person
	da(c1)
	// 没办法继续使用 da函数，实现其他类型叫
}

    package main

// 接口示例 : 不管什么牌子的车，都可以跑
// 定义一个car接口类型，不管是什么结构体，只要有run方法都能是 car类型
type car interface {
	run()
}

// 法拉利结构体
type Ferrari struct {
	brand string
}

// 为法拉利添加一个方法
func (f Ferrari) run() {
	fmt.Printf("%s速度70KM/H\n", f.brand)
}

// 保时捷结构体
type Porsche struct {
	brand string
}

// 为保时捷添加一个方法
func (p Porsche) run() {
	fmt.Printf("%s速度70KM/H\n", p.brand)
}

// drive 函数接收一个 car 类型的变量
func drive(c car) {
	c.run()
}

func main() {
	var f1 = Ferrari{
		brand: "法拉利",
	}
	var p1 = Porsche{
		brand: "保时捷",
	}
	drive(f1)
	drive(p1)
}

// 执行结果
法拉利速度70KM/H
保时捷速度70KM/H

    # 三、接口的定义

    • Go语言提倡面向接口编程；
    • 每个接口由数个方法组成，接口的定义格式如下：
    type 接口类型名 interface{
    方法名1( 参数列表1 ) 返回值列表1
    方法名2( 参数列表2 ) 返回值列表2
    …
}
    1
    2
    3
    4
    5
    • 其中：
      • 接口名：使用type将接口定义为自定义的类型名。Go语言的接口在命名时，一般会在单词后面添加er，如有写操作的接口叫Writer，有字符串功能的接口叫Stringer等。接口名最好要能突出该接口的类型含义。
      • 方法名：当方法名首字母是大写且这个接口类型名首字母也是大写时，这个方法可以被接口所在的包（package）之外的代码访问。
      • 参数列表、返回值列表：参数列表和返回值列表中的参数变量名可以省略。
    • 举个例子：
    type writer interface{
    Write([]byte) error
}
    1
    2
    3
    • 当你看到这个接口类型的值时，你不知道它是什么，唯一知道的就是可以通过它的Write方法来做一些事情;

    # 四、实现接口的条件

    • 一个对象只要全部实现了接口中的方法，那么就实现了这个接口。换句话说，接口就是一个==需要实现的方法列表==。
    • 我们来定义一个Sayer接口：
    // Sayer 接口
type Sayer interface {
	say()
}
    1

    // dog实现了Sayer接口
func (d dog) say() {
	fmt.Println("汪汪汪")

// cat实现了Sayer接口

	fmt.Println("喵喵喵")

    # 五、接口类型变量

    • 那实现了接口有什么用呢？
    • 接口类型变量能够存储所有实现了该接口的实例。 例如上面的示例中，Sayer类型的变量能够存储dog和cat类型的变量。
    func main() {
	var x Sayer // 声明一个Sayer类型的变量x
	a := cat{}  // 实例化一个cat
	b := dog{}  // 实例化一个dog
	x = a       // 可以把cat实例直接赋值给x
	x.say()     // 喵喵喵
	x = b       // 可以把dog实例直接赋值给x
	x.say()     // 汪汪汪
}

type animal interface {

	eat(string)
	// eat(s string) 	接口中定义的方法，参数名称是可以直接省略的

// 定义猫结构体
type cat struct {
	name string
	feet int8
}

// 为猫结构体添加一个移动的方法
func (c cat) move() {
	fmt.Println("猫上炕")
}

// 为猫结构体添加一个吃的方法

// 定义鸡的结构体
type chiken struct {
	feet int8
}

// 为鸡结构体增加一个移动的方法
func (c chiken) move() {
	fmt.Println("鸡动")
}

// 为鸡结构体增加一个吃的方法
func (c chiken) eat(foot string) {
	fmt.Printf("鸡吃%s\n", foot)
}

func main() {
	// 注意 ：接口是作为一种类型来使用的
	var a1 animal          // 定义一个接口类型的变量
	fmt.Printf("%T\n", a1) // <nil>
	bc := cat{             // 定义一个cat类型的变量bc
		name: "钢弹",
		feet: 4,
	}
	a1 = bc
	a1.eat("小饼干")          // 猫吃的是小饼干
	fmt.Println(a1)        // {钢弹 4}
	fmt.Printf("%T\n", a1) // main.cat	(接口保存的分为两个部分 ：值的类型和值)

	kfc := chiken{
		feet: 2,
	}
	a1 = kfc
	fmt.Printf("%T\n", a1) // main.chiken
}

    # 六、值接收者和指针接收者实现接口的区别

    • 我们有一个Mover接口和一个dog结构体。

	move()

type dog struct {}

	x = wangcai         // x可以接收dog类型
	var fugui = &dog{}  // 富贵是*dog类型

	x.move()
}

    # 6.3.1、值接收者实现接口方法

    // 接口中使用"值接收者" 和 "指针接收者"的区别 ? (方法是值接收还是指针接收)
package main

import "fmt"

// 动物接口定义
type animal interface {
	move()
	eat(string)
	// eat(s string) 	接口中定义的方法，参数名称是可以直接省略的
}

// 定义猫结构体
type cat struct {
	name string
	feet int8
}

// 为猫结构体添加一个移动的方法 -> 值接收者实现接口方法
func (c cat) move() {
	fmt.Println("猫上炕")
}

// 为猫结构体添加一个吃的方法 -> 值接收者实现接口方法
func (c cat) eat(foot string) {
	fmt.Printf("猫吃的是%s\n", foot)
}

func main() {
	// 注意 ：接口是作为一种类型来使用的
	var a1 animal // 定义一个接口类型的变量

	// -- 值 cat -- //
	bc := cat{ // 定义一个cat类型的变量bc
		name: "钢弹",
		feet: 4,
	}
	a1 = bc
	a1.eat("小饼干")   // 猫吃的是小饼干
	fmt.Println(a1) // {钢弹 4}	结构体
	// fmt.Printf("%T\n", a1) // main.cat	(接口保存的分为两个部分 ：值的类型和值)

	// -- 指针 *cat -- //
	bc1 := &cat{
		name: "土豆",
		feet: 4,
	}
	a1 = bc1
	a1.eat("小饼干")   // 猫吃的是小饼干
	fmt.Println(a1) // &{土豆 4} 结构体指针
}

// 执行结果
猫吃的是小饼干
{钢弹 4}
猫吃的是小饼干
&{土豆 4}

    # 6.3.3、总结

    • 使用值接收者实现接口与使用指针接收者实现接口的区别 ？
      • 1、使用值接收者实现接口 : 结构体类型和结构体指针类型的变量都能存；
      • 2、使用指针接收者实现接口 : 只能存储结构体指针类型的变量；

package main

import "fmt"

// 动物接口定义
type animal interface {
	move()
}

// 定义哺乳类接口
type buru interface {
	eat(string)
	// eat(s string) 	接口中定义的方法，参数名称是可以直接省略的
}

// 定义猫结构体
type cat struct {
	name string
	feet int8
}

// ---- cat 实现了animal 和 buru 接口  ---- //
// 为猫结构体添加一个移动的方法 -> 指针接收者实现接口方法
func (c *cat) move() {
	fmt.Println("猫上炕")
}

// 为猫结构体添加一个吃的方法 -> 指针接收者实现接口方法
func (c *cat) eat(foot string) {
	fmt.Printf("猫吃的是%s\n", foot)
}

func main() {
	// 注意 ：接口是作为一种类型来使用的
	var a1 animal // 定义一个接口类型的变量
	var b1 buru

	// -- 指针 *cat -- //
	bc1 := &cat{
		name: "土豆",
		feet: 4,
	}
	a1 = bc1
	b1 = bc1

	a1.move()
	b1.eat("小熊饼干")

}

    # 十、类型断言

    • 空接口可以存储任意类型的值，那我们如何获取其存储的具体数据呢？

    # 10.1、接口值

    • 一个接口的值(简称接口值)是由 ==一个具体类型== 和 ==具体类型的值== 两部分组成，这两部分分别称为接口的 ==动态类型== 和 ==动态值==；
    • 我们来看一个具体例子：
    var w io.Writer
w = os.Stdout
w = new(bytes.Buffer)
w = nil
    1
    2
    3
    4
    • 请看下图分解：

    • 想要判断空接口中的值这个时候就可以使用类型断言，其语法格式：
    x.(T)
    1
    • 其中：
      • x：表示类型为interface{}的变量
      • T：表示断言x可能是的类型。
    • 该语法返回两个参数，第一个参数是x转化为T类型后的变量，第二个值是一个布尔值，若为true则表示断言成功，为false则表示断言失败。
    • 示例：
    func main() {
	var x interface{}
	x = "Hello 沙河"
	v, ok := x.(string)
	if ok {
		fmt.Println(v)
	} else {
		fmt.Println("类型断言失败")
	}
}

	getAge() int
	talk()

	name string
	age  int

	return p.age
}

	fmt.Printf(">>> 大家好，我的名字叫%s,我今年%d岁\n", p.getName(), p.getAge())
}

func doIntroduce(p People) {
	fmt.Println("老板：接下来是一个学生吗？")
	if t, ok := p.(*Student); ok {
		fmt.Printf("助理：是的，他已经通过了我们的学生认证。这是他的基础信息: %v\n", t)
		fmt.Printf("老板：好的，%s同学，请开始\n", t.getName())
	}
	p.talk()
}

func main() {
	xiaokai := Student{"小凯", 18}
	doIntroduce(&xiaokai)
